# V2 故障模拟开发计划

## 目标
实现六类可重复触发的故障场景，以及日志、指标、链路、发布和事故查询接口；不实现 Agent 和复杂前端。

## 阶段
- [x] 阶段 1：核对 V1 模型与项目结构
- [x] 阶段 2：补充查询与场景响应模型
- [x] 阶段 3：实现六类故障场景造数服务
- [x] 阶段 4：实现 scenarios 和证据查询 API
- [x] 阶段 5：执行初始化、六场景和查询接口端到端验证

## 约束与决策
- 使用现有 SQLAlchemy 模型，不引入 Agent 或前端变更。
- 每次场景调用创建独立 trace_id，并在日志、链路和响应中关联。
- 场景写入在单个数据库事务中完成。
- 查询结果默认按业务时间倒序返回。
- 场景执行前校验基础服务数据；未初始化时返回 HTTP 409。

## 验证结果
- FastAPI 成功注册 16 个 `/api` 路由。
- 六个场景均通过真实 HTTP 调用创建数据并返回关键证据。
- 日志关键词、指标筛选、链路详情、发布列表和事故详情查询通过。
- 慢 SQL 最大 span、连接池高使用率和关键词证据通过数据库定向校验。
